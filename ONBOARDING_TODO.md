# Онбординг — сценарии для проверки

## ✅ Проверено
- Создание кошелька (полный путь)
- Генерация seed: loading-состояние, блокировка кнопки
- Верификация seed (правильная фраза)
- Экран пароля → SKIP
- Реферальный код → SKIP
- Главный экран после онбординга

---

## ❌ Не проверено

### 1. RESTORE WALLET
- Поля ввода, валидация
- Что показывается при неверном seed
- Переход на главный экран после восстановления

### 2. SET PASSWORD (не SKIP)
- Установка пароля при первом запуске
- Экран ввода пароля при повторном открытии приложения
- Биометрика (отпечаток / Face ID)
- Что если пароль забыт

### 3. Неверный seed при верификации (CheckingNewWallet)
- Текст ошибки — StringEnum.SEED_DOES_NOT_MATCH — хардкод или ресурс?
- Локализован ли он на RU?
- Можно ли повторить ввод без перезапуска

### 4. Ошибка сети при генерации кошелька
- Кнопка включается с текстом "Wrong, restart please."
- Но кнопка по-прежнему называется "I'VE SAVED IT" — вводит в заблуждение
- Нет явного сигнала "нажми чтобы повторить"

### 5. Root-диалог на реальных устройствах
- Кнопка "Выйти" закрывает приложение
- Xiaomi, OnePlus, Pixel с кастомными прошивками могут срабатывать
- Проверить насколько агрессивна проверка root
