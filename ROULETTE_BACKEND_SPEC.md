# Спецификация бэкенда: Рулетка на XRP

> Версия: 1.0  
> Дата: 2026-06-14  
> Сеть: XRPL Mainnet (`wss://xrplcluster.com`)

---

## 1. Адреса и параметры

| Параметр | Значение |
|----------|----------|
| Адрес сервера (входящие ставки) | `rGrEJZaBFYhPGuyM7NiJbJw2yXVB9vJHah` |
| Минимальная одиночная ставка | **0.1 XRP** |
| Максимальная одиночная ставка | **100 XRP** |
| Тип рулетки | Европейская (0–36, одно зеро) |

---

## 2. Входящая транзакция — ставка игрока

### 2.1 Структура XRPL-транзакции

```json
{
  "transaction": {
    "TransactionType": "Payment",
    "Account": "<адрес кошелька игрока>",
    "Destination": "rGrEJZaBFYhPGuyM7NiJbJw2yXVB9vJHah",
    "Amount": "<сумма в дропах>",
    "Memos": [
      {
        "Memo": {
          "MemoData": "<HEX-строка>"
        }
      }
    ]
  }
}
```

### 2.2 Декодирование MemoData

`MemoData` — это UTF-8 строка, закодированная в HEX (uppercase).

```python
memo_text = bytes.fromhex(memo_data_hex).decode("utf-8")
```

**Пример:**
```
MemoData: "4245543a523a6e35403…"  →  "BET:R:n5@1.5,r@2.0,d1@0.5:ref123"
```

### 2.3 Формат мемо ставки

```
BET:R:{ставка1}@{сумма1},{ставка2}@{сумма2},...:{реферал}
```

| Часть | Значение |
|-------|----------|
| `BET` | Идентификатор команды |
| `R` | Тип игры: рулетка (Roulette) |
| `{ставкаN}@{суммаN}` | Код ставки + сумма в XRP (разделитель `,`) |
| `{реферал}` | Реферальный код игрока (`"0"` если нет) |

**Реальные примеры:**
```
BET:R:n5@1.5:ref123                          — одна ставка: число 5, 1.5 XRP
BET:R:r@2.0,b@2.0:0                          — две ставки: красные + чёрные, по 2 XRP
BET:R:n0@0.5,d1@1.0,c2@1.0,o@2.0:XURA99    — четыре ставки
```

### 2.4 Проверка входящей транзакции

Перед обработкой выполнить проверки:

1. `Destination == "rGrEJZaBFYhPGuyM7NiJbJw2yXVB9vJHah"`
2. Наличие поля `Memos[0].Memo.MemoData`
3. После декодирования мемо начинается с `BET:R:`
4. Транзакция успешно включена в леджер (`meta.TransactionResult == "tesSUCCESS"`)
5. `Amount` в дропах **=** сумме всех ставок × 1 000 000 (±1 дроп на погрешности округления)

## 3. Таблица кодов ставок

### 3.1 Прямые ставки на число

| Код | Ставка | Множитель выплаты |
|-----|--------|-------------------|
| `n0` | Число 0 | **36×** |
| `n1` – `n36` | Числа 1–36 | **36×** |

### 3.2 Внешние ставки

| Код | Ставка | Условие выигрыша | Множитель |
|-----|--------|-----------------|-----------|
| `r` | Красные | Выпало красное число | **2×** |
| `b` | Чёрные | Выпало чёрное число | **2×** |
| `o` | Нечётные | Число нечётное, ≠ 0 | **2×** |
| `e` | Чётные | Число чётное, ≠ 0 | **2×** |
| `l` | Малые (1–18) | 1 ≤ число ≤ 18 | **2×** |
| `h` | Большие (19–36) | 19 ≤ число ≤ 36 | **2×** |
| `d1` | 1-я дюжина | 1 ≤ число ≤ 12 | **3×** |
| `d2` | 2-я дюжина | 13 ≤ число ≤ 24 | **3×** |
| `d3` | 3-я дюжина | 25 ≤ число ≤ 36 | **3×** |
| `c1` | 1-я колонка | число % 3 == 1 (1,4,7,…,34) | **3×** |
| `c2` | 2-я колонка | число % 3 == 2 (2,5,8,…,35) | **3×** |
| `c3` | 3-я колонка | число % 3 == 0 (3,6,9,…,36) | **3×** |

### 3.3 Цветовая карта чисел

**Чёрные:** 2, 4, 6, 8, 10, 11, 13, 15, 17, 20, 22, 24, 26, 28, 29, 31, 33, 35  
**Красные:** все остальные 1–36 (не в списке чёрных)  
**Зелёное:** 0 (не является красным или чёрным; все внешние ставки проигрывают на 0)

## 4. Игровая логика сервера

### 4.1 Алгоритм обработки

```
1. Получить входящую транзакцию на адрес сервера
2. Декодировать MemoData из HEX → UTF-8
3. Проверить, что мемо начинается с "BET:R:"
4. Распарсить ставки (см. §5)
5. Сгенерировать выигрышное число N (0–36), криптографически случайное
6. Для каждой ставки вычислить: выиграла или нет (см. §3)
7. Вычислить итоговую выплату (см. §4.2)
8. Отправить ответную транзакцию игроку (см. §6)
```

### 4.2 Расчёт выплаты

```
total_payout = 0
for each bet in bets:
    if bet_wins(bet.code, winning_number):
        total_payout += bet.amount * multiplier(bet.code)
```

**Случай победы:** `total_payout > 0`  
**Случай проигрыша:** `total_payout == 0` (все ставки проиграли)

> ⚠️ Суммы ставок в мемо — в XRP. Для XRPL-транзакции умножить на 1 000 000 (дропы).

### 4.3 Генерация выигрышного числа

- Диапазон: **0–36 включительно** (европейская рулетка)
- Алгоритм: криптографически стойкий RNG (не `Math.random()`)
- Выигрышное число включается в ответное мемо для верификации клиентом

## 5. Парсинг мемо — псевдокод

```python
def parse_roulette_memo(memo_data_hex: str) -> dict:
    text = bytes.fromhex(memo_data_hex).decode("utf-8")
    
    # Структура: BET:R:{ставки}:{реферал}
    if not text.startswith("BET:R:"):
        return None  # не рулетка, игнорировать
    
    # Убрать префикс
    body = text[6:]  # "n5@1.5,r@2.0,d1@0.5:ref123"
    
    # Разделить ставки и реферал по последнему ":"
    last_colon = body.rfind(":")
    bets_part = body[:last_colon]   # "n5@1.5,r@2.0,d1@0.5"
    referral  = body[last_colon+1:] # "ref123"
    
    bets = []
    for token in bets_part.split(","):
        parts = token.split("@", 1)
        if len(parts) != 2:
            continue  # битый токен, пропустить
        code   = parts[0].strip()
        amount = Decimal(parts[1].strip())
        bets.append({"code": code, "amount": amount})
    
    return {"bets": bets, "referral": referral}


def bet_wins(code: str, winning_number: int) -> bool:
    BLACK = {2,4,6,8,10,11,13,15,17,20,22,24,26,28,29,31,33,35}
    
    # Прямая ставка на число
    if code.startswith("n"):
        return int(code[1:]) == winning_number
    
    # Все внешние ставки проигрывают на 0
    if winning_number == 0:
        return False
    
    n = winning_number
    if code == "r":  return n not in BLACK
    if code == "b":  return n in BLACK
    if code == "o":  return n % 2 != 0
    if code == "e":  return n % 2 == 0
    if code == "l":  return 1 <= n <= 18
    if code == "h":  return 19 <= n <= 36
    if code == "d1": return 1 <= n <= 12
    if code == "d2": return 13 <= n <= 24
    if code == "d3": return 25 <= n <= 36
    if code == "c1": return n % 3 == 1
    if code == "c2": return n % 3 == 2
    if code == "c3": return n % 3 == 0
    
    return False  # неизвестный код


MULTIPLIERS = {
    "r":2, "b":2, "o":2, "e":2, "l":2, "h":2,
    "d1":3, "d2":3, "d3":3,
    "c1":3, "c2":3, "c3":3,
}

def multiplier(code: str) -> int:
    if code.startswith("n"):
        return 36
    return MULTIPLIERS.get(code, 2)
```

## 6. Ответная транзакция сервера → игрок

### 6.1 Поля XRPL-транзакции

| Поле | Значение |
|------|----------|
| `Account` | `rGrEJZaBFYhPGuyM7NiJbJw2yXVB9vJHah` (сервер) |
| `Destination` | `Account` из входящей транзакции (адрес игрока) |
| `Amount` | **total_payout** в дропах (при проигрыше — см. §6.3) |
| `Memos[0].MemoData` | HEX-кодированный ответ (см. §6.2) |

### 6.2 Формат ответного мемо

#### Победа (хотя бы одна ставка выиграла):
```
WIN:{выигрышное_число}
```
Примеры:
```
WIN:17
WIN:0
WIN:32
```

#### Проигрыш (все ставки проиграли):
```
LOSE:{выигрышное_число}
```
Примеры:
```
LOSE:0
LOSE:24
```

#### Дополнительные команды (совместимость с другими играми):
```
JKPT:{число}   — джекпот (используется как WIN)
LOTO:{число}   — лото-выигрыш (используется как WIN)
REF:{код}      — назначение реферального кода
```

> Клиент читает `MemoData` в uppercase после декодирования — отправлять мемо можно в любом регистре.

### 6.3 Сумма при проигрыше

Клиент ожидает транзакцию **в любом случае** (иначе колесо будет крутиться до таймаута).  
При полном проигрыше отправить символическую сумму: **0.000012 XRP (12 дропов)** — минимальный размер комиссии, возврат не предусмотрен.

> Количество `delivered_amount` клиент показывает только при WIN, поэтому при LOSE сумма не важна.

### 6.4 Кодирование MemoData для ответа

```python
def to_hex(text: str) -> str:
    return text.encode("utf-8").hex().upper()

memo_data = to_hex("WIN:17")    # → "57494E3A3137"
memo_data = to_hex("LOSE:0")   # → "4C4F53453A30"
```

## 7. Таймлайн и идемпотентность

### 7.1 Допустимое время ответа

Клиент показывает крутящееся колесо и ждёт ответа **без таймаута** (ограничение — 4 мин до заголовка XRP-транзакции `LastLedgerSequence`). Рекомендуемое время ответа сервера: **3–8 секунд** (имитирует реалистичный спин).

### 7.2 Дедупликация

- Сохранять хэш входящей транзакции (`hash`) в базу до отправки ответа
- Если транзакция с тем же `hash` уже обработана — не отвечать повторно
- Поле `hash` доступно в WebSocket-сообщении типа `transaction`

### 7.3 Порядок операций

```
1. Принять транзакцию
2. Проверить: не обработана ли уже (по hash)
3. Сохранить hash как "в обработке"
4. Сгенерировать выигрышное число
5. Рассчитать выплату
6. Подписать и отправить ответную транзакцию
7. Сохранить результат: {hash, winning_number, payout, player_address}
```

## 9. Примеры полного цикла

### Пример 1 — одиночная ставка, победа

```
Игрок отправляет:
  Amount:   1 500 000 дропов  (1.5 XRP)
  MemoData: HEX("BET:R:n17@1.5:0")

Сервер генерирует: winning_number = 17

Расчёт:
  n17 wins? → 17 == 17 → ДА
  payout = 1.5 × 36 = 54.0 XRP

Сервер отвечает:
  Amount:   54 000 000 дропов  (54.0 XRP)
  MemoData: HEX("WIN:17")
```

### Пример 2 — несколько ставок, частичная победа

```
Игрок отправляет:
  Amount:   5 000 000 дропов  (5.0 XRP)
  MemoData: HEX("BET:R:n5@1.5,r@2.0,d1@1.5:XURA99")

Сервер генерирует: winning_number = 5

Расчёт:
  n5  wins? → 5 == 5       → ДА → 1.5 × 36 = 54.0 XRP
  r   wins? → 5 = красное  → ДА → 2.0 × 2  = 4.0 XRP
  d1  wins? → 1≤5≤12       → ДА → 1.5 × 3  = 4.5 XRP
  total_payout = 54.0 + 4.0 + 4.5 = 62.5 XRP

Сервер отвечает:
  Amount:   62 500 000 дропов  (62.5 XRP)
  MemoData: HEX("WIN:5")
```

### Пример 3 — несколько ставок, полный проигрыш

```
Игрок отправляет:
  Amount:   3 000 000 дропов  (3.0 XRP)
  MemoData: HEX("BET:R:r@2.0,d1@1.0:0")

Сервер генерирует: winning_number = 0   (зеро)

Расчёт:
  r   wins? → 0 = зеро → НЕТ (внешние ставки на 0 проигрывают)
  d1  wins? → 0 < 1    → НЕТ
  total_payout = 0

Сервер отвечает:
  Amount:   12 дропов  (минимум, символически)
  MemoData: HEX("LOSE:0")
```

## 10. Граничные случаи и обработка ошибок

| Ситуация | Действие сервера |
|----------|-----------------|
| `Amount` не совпадает с суммой ставок в мемо | Отыграть ставки пропорционально ИЛИ вернуть деньги с мемо `ERR:AMOUNT_MISMATCH` |
| Неизвестный код ставки | Пропустить эту ставку, отыграть остальные |
| Пустой список ставок после парсинга | Вернуть деньги с мемо `ERR:NO_BETS` |
| Ставка < 0.1 XRP | Не отыгрывать эту ставку, вернуть её сумму в рамках LOSE |
| Ставка > 100 XRP | Принять как 100 XRP, остаток вернуть или отказать целиком |
| Мемо не начинается с `BET:R:` | Игнорировать (другой тип игры или прямой перевод) |
| Транзакция уже обработана (дубль) | Игнорировать |

## 11. Хранение данных (минимально необходимое)

```sql
CREATE TABLE roulette_rounds (
    id              BIGSERIAL PRIMARY KEY,
    tx_hash         VARCHAR(64) UNIQUE NOT NULL,   -- hash входящей транзакции
    player_address  VARCHAR(40) NOT NULL,
    bets_memo       TEXT NOT NULL,                 -- исходный мемо текст
    winning_number  SMALLINT NOT NULL,             -- 0–36
    total_bet_xrp   DECIMAL(20,6) NOT NULL,
    total_payout_xrp DECIMAL(20,6) NOT NULL,       -- 0 если проигрыш
    referral        VARCHAR(50),
    created_at      TIMESTAMP DEFAULT NOW()
);
```
