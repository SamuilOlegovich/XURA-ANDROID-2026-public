# XURA — Дизайн-ревью приложения

> Режим: Дизайнер · Дата: 14.06.2026 · Экраны: 32 layout · Тема: тёмная

---

## 1. Дизайн-система

### Цветовая палитра

| Токен | Hex | Назначение |
|-------|-----|------------|
| `xura_black` | `#000000` | Фон всех экранов |
| `xura_surface` | `#08081A` | Primary container |
| `xura_card` | `#0D0D20` | Карточки, MaterialCardView |
| `xura_cyan` | `#00D4FF` | Primary акцент, интерактив |
| `xura_purple` | `#9020D0` | Secondary акцент |
| `xura_gold` | `#FFB000` | Tertiary, победы, история |
| `xura_pink` | `#FF2080` | Деструктивные / азартные действия |
| `xura_error` | `#FF3060` | Ошибки |
| `xura_success` | `#00FF88` | Успех (не используется в layouts) |

**Вердикт по палитре: 8/10.** Фирменный градиент (cyan→blue→indigo→purple→magenta→pink→orange→gold) создаёт сильную Brand Identity. Чистый чёрный фон (#000000) даёт глубину и хорошо смотрится на OLED. Проблема: `xura_success` (#00FF88) нигде не задействован в разметке — токен объявлен, но не используется.

### Типографика

- Шрифт: **Montserrat** — используется на 100% экранов. Консистентно.
- Иерархия (по факту из layouts):

| Роль | Размер | Стиль |
|------|--------|-------|
| Hero / Win | 60sp | Bold |
| Screen title | 28sp | Bold |
| Game title | 24sp | Bold |
| Card title | 15–17sp | Bold |
| Subtitle | 11sp | Regular |
| Label / meta | 9–11sp | AllCaps + LetterSpacing |

**Вердикт: 7/10.** Иерархия чёткая, но 60sp для "BET WON" на маленьких экранах (< 5") обрезается. MD3 рекомендует Display Large = 57sp. 9sp для меток баланса — ниже минимально читаемого размера (12sp по AA).

### Отступы и сетка

- Горизонтальные поля: **24dp** — соответствует MD3
- Между карточками: **12dp** — кратно 4dp ✓
- Bottom nav отступ снизу: **16dp** ✓
- Logo горизонтальные поля: **80dp** — слишком широкие, логотип зажат

**Вердикт: 8/10.** Кратность 4dp соблюдается. Исключение — 40dp отступ в Settings (marginTop: 40dp) — некратно стандарту MD3 (лучше 32dp или 48dp).

## 3. Детальный анализ по экранам

---

### 🎬 Splash Screen (`splash_screen.xml`)

**Что хорошо:**
- Минималистичный — только логотип на чёрном. Профессионально.
- Логотип центрирован правильно (top+bottom constraint).

**Проблемы:**
- Нет никакого loading-индикатора. Пользователь не знает, что приложение загружается.
- Нет состояния ошибки (нет сети, нет кошелька).
- Логотип с полями 80dp с каждой стороны — итого логотип занимает 50% ширины экрана. На узких экранах это нормально, но на планшете выглядит слабо.

**Рекомендация:** Добавить тонкий CircularProgressIndicator (3dp, xura_cyan) под логотипом.

---

### 🏠 Restore or Create Wallet (`restore_or_create_new_wallet_page.xml`)

**Что хорошо:**
- Два чётких CTA в нижней части экрана — правильная иерархия снизу вверх (первичный → вторичный).
- Карточки 80dp с иконкой справа и subtitle снизу — единый паттерн выдержан.
- Gold для "Create", Cyan для "Restore" — логичная дифференциация.

**Проблемы:**
- Огромное пустое пространство между логотипом и кнопками. Нет никакого контекста/иллюстрации/описания.
- Нет заголовка экрана (что это за экран — непонятно без подсказок).
- marginBottom="40dp" у нижней карточки — некратно 8dp (лучше 40dp→40dp, но MD3 предпочитает 32dp или 48dp).

---

### 💼 Main (Wallet) Screen (`activity_main.xml`)

**Что хорошо:**
- Логотип → баланс → 3 action cards — классическая вертикальная иерархия.
- CircularProgressIndicator для баланса (invisible/visible toggle) — правильный паттерн.
- Цветовая дифференциация карточек: pink (SEND), cyan (REQUEST), gold (HISTORY) — интуитивно.
- Floating pill Bottom Nav с 16dp отступами.

**Проблемы:**
- `textSize="9sp"` для label "YOUR BALANCE" — слишком мелко, ниже порога читаемости AA.
- `textSize="40sp"` для суммы — хорошо, но нет единицы измерения (XRP) в самом TextView — она появляется из кода, что может вызвать FOUC.
- Bottom Nav: `app:labelVisibilityMode="unlabeled"` — нет текстовых подписей. MD3 **не рекомендует** полностью убирать labels для Nav Bar (иконки + labels = меньше когнитивная нагрузка).
- Logo margin Start/End = 80dp создаёт очень узкий логотип.

---

### 🎮 Select Game Page (`select_game_page.xml`)

**Что хорошо:**
- Три игры представлены в одинаковых карточках с уникальными цветами: pink, cyan, gold.
- Декоративные иконки (игральные карты для Color, числа для Number, колесо для Roulette) добавляют характер.
- Subtitle под каждым тайтлом объясняет механику.

**Проблемы:**
- Иконки разного размера: Color = 56×44dp FrameLayout, Number = 68×50dp, Roulette = 56×56dp ImageView. Нет единообразия.
- Карточки выровнены снизу вверх (все `constraintBottom_to`), что означает сначала рендерится нижний элемент. Логичнее строить сверху вниз с chainStyle="packed".
- "ROULETTE" карточка не показывает статус "COMING SOON" визуально — нужен badge или opacity 60% для недоступной игры.

---

### 🎨 Guess the Color Game (`guess_the_color_game_page.xml`)

**Что хорошо:**
- Поле ввода ставки с suffix "XRP" и правильным TextInputLayout.OutlinedBox стилем.
- Chip группа для быстрых сумм (1, 5, 10 XRP) — отличный UX паттерн.
- Две карточки RED/BLACK с разными фонами — визуально понятно.

**Проблемы:**
- **Критично:** Баланс задан hardcoded: `android:text="97,456782  XRP"` — это значение не из ресурсов, останется если data binding не сработает.
- `textSize="16sp"` для баланса — слишком мелко для hero-числа в игровом экране.
- Поле ввода прикреплено снизу к chip-группе, которая прикреплена снизу к RED-кнопке. Это инверсия порядка восприятия (сначала кнопки, потом ввод) — нелогично.
- `your_balance_text_view`: `height="16dp"` — фиксированная высота для TextView это антипаттерн (при увеличении шрифта системой будет обрезан).
- Нет loading-состояния при ожидании результата ставки.
- Нет error-состояния отдельного от TextInputLayout.

---

### 🎰 Roulette Game (`roulette_game_page.xml`)

**Что хорошо:**
- ScrollView позволяет работать с большим столом.
- Rules + Balance в одной строке (LinearLayout horizontal) — компактно.
- MaterialCardView для стола с правильными атрибутами (stroke, cornerRadius).

**Проблемы:**
- Рулеточный стол генерируется программно (Java-код) — нет никакого контроля над UI из XML. Это означает что стол может выглядеть плохо в разных размерах экрана.
- Нет состояния "загрузка" и "ошибка" для стола.
- SPIN-кнопка использует `bg_card_history` (золотой стиль для "истории транзакций") — запутывает семантику. Нужен отдельный стиль для action-кнопки.
- Стол использует `layoutMarginStart="12dp"` вместо стандартных 24dp — нарушает сетку.
- Нет индикатора текущей выбранной ячейки при прокрутке вниз (нет sticky header).

---

### ⚙️ Settings Page (`settings_page.xml`)

**Что хорошо:**
- NestedScrollView с LinearLayout — правильный паттерн для длинного списка.
- Карточка тестового режима (card_test_balance) с gold-обводкой и MaterialCardView — хорошо.
- Reset-кнопка в карточке теста — логично расположена внутри контекста.
- Все карточки 80dp с единым паттерном title+subtitle+icon.

**Проблемы:**
- Карточка SET PASSWORD использует `bg_card_send` (розовый стиль) — семантически "send payment" ≠ "set password". Нужен отдельный стиль для секьюрити.
- GAME MODE использует `bg_card_history` (золотой) — неочевидная связь.
- 3 нижних карточки (Language, Become Referral, Info) имеют одинаковый стиль `bg_card_glass_clickable` — не дифференцированы по важности.
- Нет секций/разделителей между группами настроек (Security / Game / About).
- `settings_text_view`: marginTop="40dp" — некратно 8dp (MD3 предпочитает 32dp или 48dp).

---

### 🏆 Win Page (`win_page.xml`)

**Что хорошо:**
- "BET WON" 60sp золотым — эмоционально сильный момент победы.
- Marquee для суммы выигрыша — решает проблему длинных сумм.
- Два CTA: PLAY AGAIN (cyan) + BACK TO GAMES (glass) — правильная иерархия.

**Проблемы:**
- 60sp textSize может быть обрезан на экранах < 5" — проверить на Pixel 4a.
- marginTop="40dp" между логотипом и "BET WON" — маловато для hero-экрана. 32dp между лого и big number.
- marginTop="40dp" для суммы (win_page_text_view_tow) — неожиданно большой разрыв.
- Нет анимации: конфетти, свечение, пульсация — момент победы плоский визуально.
- `congratulations` (28sp) и `bet_won` (60sp) стоят друг под другом оба cyan — нет цветового контраста между ними.

## 4. Главные проблемы

### ❌ Критические

1. **Hardcoded значение баланса** в `guess_the_color_game_page.xml`: `android:text="97,456782  XRP"` — если ViewModel не отработает, пользователь увидит чужой баланс.

2. **Bottom Nav без labels**: `labelVisibilityMode="unlabeled"` — пользователи без опыта не поймут иконки. MD3 требует минимум labeled при 3 items.

3. **9sp текст** для "YOUR BALANCE" label — ниже минимального порога читаемости (12sp). Нарушение WCAG AA.

4. **Фиксированные высоты TextView** (`height="16dp"`, `height="20dp"`, `height="28dp"`) — при системном увеличении шрифта текст будет обрезан. Нужен `wrap_content`.

5. **Отсутствие loading и error состояний** на большинстве игровых экранов.

### ⚠️ Важные

6. **Логотип 80dp с каждой стороны** — на широких экранах логотип выглядит слишком маленьким. Рекомендую 60–72dp или `maxWidth="200dp"`.

7. **Семантический мисматч фонов карточек**: `bg_card_send` (розовый) используется для "Set Password", `bg_card_history` (золотой) — для "Spin" и "Game Mode". Нужна семантическая разбивка: security-card, action-card, nav-card.

8. **Roulette SPIN**: `bg_card_history` — не тот стиль. Spin — это primary action, а не navigation.

9. **Settings без секций**: 6 карточек в ряд без группировки = нет информационной архитектуры. Нужны header-labels (SECURITY / GAME / ACCOUNT / APP).

10. **Win Page без анимации**: экран победы — ключевой момент эмоции. Нет конфетти, нет анимации появления числа. Плоский опыт.

### 💡 Улучшения

11. **Splash screen**: добавить subtle shimmer или тонкий progress indicator.

12. **Restore/Create screen**: добавить иллюстрацию или описание "что такое XURA" между лого и кнопками — экран сейчас пустой.

13. **Roulette COMING SOON**: карточка доступна для нажатия, но игра не готова. Нужен disabled-state или badge.

14. **Cards touch feedback**: убедиться, что ripple (`@color/xura_glass_fill`) виден на всех карточках — стекловидный ripple на тёмном стекле может быть невидим.

15. **Info секции** (info_main_page, become_referral): стены текста без разбивки. Нужны иконки для каждого пункта, правильные отступы между блоками.

## 5. Что сделано хорошо

1. **Фирменная дизайн-система** — XURA palette чёткая, узнаваемая, последовательно применена.
2. **Единый card-pattern** — title + subtitle + icon справа — работает на всех экранах.
3. **Montserrat** — качественный шрифт, правильная жирность для тёмного фона.
4. **Material3 компоненты**: TextInputLayout.OutlinedBox, ChipGroup, MaterialCardView, BottomNavigationView — всё правильно.
5. **Bottom Nav floating pill** — современный паттерн, хорошо выглядит.
6. **CircularProgressIndicator** для баланса — правильная обработка loading-состояния.
7. **contentDescription** на всех интерактивных элементах — внимание к accessibility.
8. **letterSpacing** на кнопках и labels — профессиональная типографика.
9. **Quick bet chips** (1/5/10 XRP) — отличный UX, сокращает время ввода.
10. **Marquee для длинных значений** — элегантное решение для сумм выигрыша.
