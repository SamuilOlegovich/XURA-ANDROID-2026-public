# Slot Machine — Техническая спецификация

Документ описывает полную логику слот-машины в приложении XURA Android.
Назначение: воспроизвести идентичное поведение на iOS.

---

## 1. Символы

Слот-машина использует **7 символов** с фиксированными ID:

| ID | Название | Метка   | Множитель ставки |
|----|----------|---------|-----------------|
| 0  | XRP      | XRP     | ×2              |
| 1  | Rocket   | RKT     | ×5              |
| 2  | Moon     | MOON    | ×10             |
| 3  | Diamond  | DIA     | ×20             |
| 4  | Whale    | WHAL    | ×50             |
| 5  | Jackpot  | JKPT    | ×250            |
| 6  | Wild     | WILD    | —               |

**Wild (6)** замещает любой символ при определении победы. Линия из трёх Wild победой не считается.

---

## 2. Рил-стрип (Reel Strip)

### Длина и распределение

Каждый барабан использует **84-позиционный стрип** (сумма весов всех символов):

| Символ  | ID | Количество | Доля   |
|---------|----|-----------|--------|
| XRP     | 0  | 30        | 35.7%  |
| Rocket  | 1  | 20        | 23.8%  |
| Moon    | 2  | 15        | 17.9%  |
| Diamond | 3  | 10        | 11.9%  |
| Whale   | 4  | 5         | 6.0%   |
| Wild    | 6  | 3         | 3.6%   |
| Jackpot | 5  | 1         | 1.2%   |

### Базовый массив (84 элемента)

```
BASE[84] = {
  0, 1, 0, 2, 0, 1, 0, 3, 0, 1,   // позиции 0–9
  2, 0, 1, 0, 4, 2, 0, 1, 3, 0,   // позиции 10–19
  1, 2, 0, 6, 1, 0, 3, 2, 0, 1,   // позиции 20–29
  0, 2, 3, 0, 1, 4, 2, 0, 1, 3,   // позиции 30–39
  0, 2, 1, 0, 5, 2, 0, 3, 1, 0,   // позиции 40–49
  6, 2, 0, 1, 3, 0, 2, 4, 1, 0,   // позиции 50–59
  3, 2, 0, 1, 0, 2, 6, 3, 0, 1,   // позиции 60–69
  2, 0, 4, 1, 0, 3, 2, 1, 0, 4,   // позиции 70–79
  1, 0, 1, 0                        // позиции 80–83
}
```

### Три барабана — три стрипа

Все три барабана используют **одну ленту**, сдвинутую на разное количество позиций:

| Барабан      | Стрип             | Сдвиг |
|--------------|-------------------|-------|
| Левый (col 0)   | LEFT = BASE       | 0     |
| Центральный (col 1) | CENTER = rotate(BASE, 28) | +28 |
| Правый (col 2)  | RIGHT = rotate(BASE, 56)  | +56 |

Функция `rotate(src, by)`:
```
rotate[i] = src[(i + by) % 84]
```

Это обеспечивает **одинаковую статистическую вероятность** символов на всех барабанах при разном визуальном порядке.

## 3. Матрица 3×3

Барабан показывает **3 ячейки** одновременно (верх / середина / низ).
При стоп-позиции `p` на барабане видны позиции:

```
row 0 (верх)   = strip[(p - 1 + 84) % 84]
row 1 (середина) = strip[p]
row 2 (низ)    = strip[(p + 1) % 84]
```

Итоговая сетка — матрица `[3][3]`:

```
         col 0 (LEFT)   col 1 (CENTER)   col 2 (RIGHT)
row 0:   верхний ряд
row 1:   средний ряд   ← главная линия
row 2:   нижний ряд
```

Построение матрицы из трёх стоп-позиций:
```
buildMatrix(stopLeft, stopCenter, stopRight):
  for row in 0..2:
    m[row][0] = LEFT  [(stopLeft   + row - 1 + 84) % 84]
    m[row][1] = CENTER[(stopCenter + row - 1 + 84) % 84]
    m[row][2] = RIGHT [(stopRight  + row - 1 + 84) % 84]
```

## 4. Линии выплат (Paylines)

Пять линий, каждая проходит через три ячейки сетки:

| № | Название         | Ячейки (row, col)              | Цвет     |
|---|------------------|--------------------------------|----------|
| 0 | Средняя горизонталь | (1,0) → (1,1) → (1,2)      | Золото `#FFB000`   |
| 1 | Верхняя горизонталь | (0,0) → (0,1) → (0,2)      | Голубой `#00D4FF`  |
| 2 | Нижняя горизонталь  | (2,0) → (2,1) → (2,2)      | Лаймовый `#40FF90` |
| 3 | Диагональ ↘         | (0,0) → (1,1) → (2,2)      | Оранжевый `#FF6A00`|
| 4 | Диагональ ↗         | (2,0) → (1,1) → (0,2)      | Малиновый `#FF40C0`|

### Логика определения победы на линии

```
resolveWin(sym0, sym1, sym2):
  found = -1
  for each sym in [sym0, sym1, sym2]:
    if sym == 6 (Wild): continue   // Wild замещает любой
    if found == -1: found = sym
    else if found != sym: return -1  // два разных не-Wild → нет победы
  return found   // -1 если все три Wild → тоже нет победы
```

Выплата считается по символу `found` × `MULTIPLIERS[found]` × размер ставки.

## 5. Протокол сервера

### Формат Memo в XRPL-транзакции

Ответ сервера приходит как `MemoData` в hex-кодировке в транзакции XRPL.
После декодирования из hex в UTF-8 и приведения к upper case:

```
SLOT:{p0},{p1},{p2}:{WIN|LOSE}
```

Примеры:
```
SLOT:12,45,67:WIN
SLOT:0,28,56:LOSE
```

| Поле | Описание |
|------|----------|
| `{p0}` | Стоп-позиция левого барабана, целое число 0–83 |
| `{p1}` | Стоп-позиция центрального барабана, 0–83 |
| `{p2}` | Стоп-позиция правого барабана, 0–83 |
| `WIN/LOSE` | Итог, определённый сервером |

**Сервер обязан использовать тот же стрип** — `BASE` с теми же сдвигами для CENTER (+28) и RIGHT (+56), иначе матрица на клиенте и сервере не совпадут.

Сервер бросает `random(0, 83)` для каждого барабана независимо, затем проверяет выигрышные линии и формирует `WIN` или `LOSE`.

## 6. Состояния экрана SlotFlasher

### 6.1 Открытие экрана (`onCreate`)

1. Все три барабана начинают **непрерывный спин** (бесконечная прокрутка).
2. Запускается звук `roulette_spin` в зацикленном режиме.
3. Запускается **таймаут ставки** (настраивается в настройках, по умолчанию 120 секунд).
4. В фоне запускается `FlasherRun` — прослушивает XRPL-сокет.
5. В TRIAL-режиме (`IS_REAL_GAME_MODE == false`) параллельно запускается `NotifierRunForTrialGame` — симулирует ответ сервера локально.

Начальный визуальный порядок (только для эффекта вращения, без привязки к стрипу):
```
leftReel:   [0, 2, 4, 1, 5, 3, 6]
centerReel: [3, 0, 5, 2, 6, 1, 4]
rightReel:  [5, 1, 3, 6, 0, 4, 2]
```

### 6.2 Получение ответа сервера (вызов `stopGame`)

Вызывается из `NotifierRun.responseToBetSlot()` на UI-потоке:

```
stopGame(text: String, win: Boolean):
  if resultShown: return     // защита от двойного вызова
  resultShown = true
  waitingForResult = false
  cancelBetTimeout()
  stopReelsSequentially(text, win)
```

### 6.3 Остановка барабанов (`stopReelsSequentially`)

1. **Построить матрицу 3×3** из `STOP_POSITIONS` (сервер) → `SlotReelStrip.buildMatrix(p0, p1, p2)`
   - Если `STOP_POSITIONS == null` — использовать `RESULT_MATRIX` (trial-режим)
   - Если оба null — полный fallback (случайная матрица, без стрипа)
2. **Для каждого барабана** построить визуальный `reelOrder[7]` через `buildVisualOrder(top, mid, bot)`.
3. Установить `reelOrder` в барабан.
4. Останавливать барабаны **слева направо с задержкой 400 мс**: левый → (400ms) → центральный → (400ms) → правый.
5. Команда остановки: `stopAtPosition(1)` — барабан плавно останавливается на **индексе 1** визуального порядка, то есть на `mid`-символе.

```
buildVisualOrder(top, mid, bot) → [7]int:
  order[0] = top
  order[1] = mid
  order[2] = bot
  // заполнить позиции 3..6 символами, которых нет в top/mid/bot
  // (обработка дублей: placed[sym] = true для каждого из top/mid/bot)
```

После остановки правого барабана — задержка 200 мс → вызов `showResult`.

### 6.4 Показ результата (`showResult`)

**При победе:**
- Подсветить среднюю строку всех трёх барабанов (`setHighlightMiddle(true)`)
- Через 350 мс запустить `paylineView.showWinLines(matrix)`
- Заголовок: градиент золото/оранжевый
- Конфетти сверху вниз (Konfetti)

**При поражении:**
- Заголовок: градиент красный/тёмный
- Пепел снизу вверх (Konfetti в тёмных тонах)

**В обоих случаях:**
- Текст с суммой выигрыша / сообщением о проигрыше
- Кнопка «Назад»
- Таймер обратного отсчёта 10 секунд → автовозврат на предыдущий экран

### 6.5 Таймаут ставки

Если за настроенное время (по умолчанию 120 сек) ответ сервера не пришёл:
- Барабаны останавливаются мгновенно (`cancelAnim`)
- Показывается сообщение «Ожидание истекло»
- Обратный отсчёт 10 секунд → автовозврат

## 7. Кросс-экранная доставка результата (SlotResult)

### Когда срабатывает

Если ответ сервера пришёл **после того, как игрок покинул** экран `SlotFlasher`:
- `VISIBLE_ON_SCREEN == false`
- Открывается отдельный экран `SlotResult`

### Передача данных (статические поля)

Данные копируются в статические поля `SlotResult` **до** асинхронной отправки `NavigationEvent`:

```
stops = SlotFlasher.STOP_POSITIONS?.clone()   // важно: клонировать!
SlotResult.PENDING_STOPS = stops
SlotResult.MASSAGE = messageText
SlotResult.IS_WIN = (outcome == WIN)
// затем postValue(NavigationEvent.SLOT_RESULT)
```

### Поведение SlotResult

1. Начинает краткий спин (те же начальные `reelOrder`, что у `SlotFlasher`).
2. Через **1100 мс** (`SPIN_BEFORE_STOP_MS`) — вызывает `revealResult()`.
3. `revealResult()` строит матрицу из `PENDING_STOPS`, устанавливает `reelOrder`, останавливает барабаны через `stopAtPosition(1)` (идентично `SlotFlasher`).
4. После остановки — `showResultUi()`: те же эффекты что в `SlotFlasher.showResult()`.
5. Кнопка «Назад», обратный отсчёт 10 сек.

## 8. Анимация выигрышных линий (SlotPaylineView)

Прозрачный View-оверлей, наложенный поверх сетки 3×3.

### Запуск

```
paylineView.showWinLines(matrix [3][3]):
  для каждой из 5 paylines:
    извлечь три символа из matrix по координатам линии
    вызвать resolveWin(sym0, sym1, sym2)
    если результат >= 0: добавить линию в список активных
  
  запустить анимации последовательно:
    каждая линия — 580 мс
    задержка между линиями — 640 мс
    интерполятор: AccelerateDecelerate
```

### Отрисовка каждой линии

1. **Свечение** — широкая полупрозрачная линия (stroke width = 24% высоты ячейки, alpha ~19%)
2. **Линия** — тонкая яркая (stroke width = 5.5% высоты ячейки)
3. **Трассировка** — линия рисуется не целиком, а по мере `progress` через `PathMeasure.getSegment()`
4. **Точки** на узлах (появляются по мере продвижения линии)
5. **Подпись** — badge с тёмным фоном-пилюлей, появляется на последних 28% анимации

Формат подписи: `"XRP ×2"`, `"JKPT ×250"` и т.д.

Положение подписи (смещение от центра линии, в долях ширины/высоты ячейки):

| Линия | dx      | dy      |
|-------|---------|---------|
| Средняя | 0     | −0.38   |
| Верхняя | 0     | +0.32   |
| Нижняя  | 0     | −0.32   |
| Диаг ↘  | −0.40 | −0.32   |
| Диаг ↗  | +0.40 | −0.32   |

### Сброс

`paylineView.reset()` — вызывается в `onPause` и `onBackPressed`. Отменяет все анимации, очищает список активных линий.

## 9. Режимы игры

| Режим | Описание |
|-------|----------|
| **LIVE** (`IS_REAL_GAME_MODE == true`) | Реальные ставки XRP через XRPL-сокет. Ответ приходит от сервера через `NotifierRun`. |
| **TRIAL** (`IS_REAL_GAME_MODE == false`) | Симуляция локально через `NotifierRunForTrialGame`. Логика генерации результата идентична серверной. |

---

## 10. Lifecycle-правила

| Событие | Действие |
|---------|----------|
| `onResume` | `VISIBLE_ON_SCREEN = true`. Если результат уже пришёл, но UI не показан — мгновенно восстановить барабаны и показать результат (`snapToPosition(1)`). |
| `onPause` | `VISIBLE_ON_SCREEN = false`. Остановить все анимации и звук, сбросить payline-оверлей. |
| `onDestroy` | Очистить `SLOT_FLASHER = null`, `STOP_POSITIONS = null`, `RESULT_MATRIX = null`. |
| `onBackPressed` | Отменить таймер, обратный отсчёт, анимации, звук. |

## 11. Краткая схема потока данных

```
Пользователь делает ставку
        │
        ▼
SlotFlasher.onCreate()
  ├── startSpin() ── все три барабана крутятся
  ├── startBetTimeout(120s)
  └── FlasherRun → слушает XRPL-сокет

        │  ответ XRPL-сокета
        ▼
NotifierRun.responseToBet()
  parses Memo: "SLOT:{p0},{p1},{p2}:{WIN|LOSE}"
        │
        ├── VISIBLE_ON_SCREEN == true?
        │         │
        │         ▼ да
        │   SlotFlasher.stopGame(text, win)
        │     buildMatrix(p0,p1,p2)
        │     buildVisualOrder × 3
        │     stopAtPosition(1) ×3 (слева направо, 400ms)
        │     showResult() → paylineView.showWinLines()
        │
        └── VISIBLE_ON_SCREEN == false?
                  │
                  ▼ нет (игрок ушёл)
            SlotResult.PENDING_STOPS = stops.clone()
            SlotResult.IS_WIN = win
            NavigationEvent.SLOT_RESULT
                  │
                  ▼
            SlotResult.onCreate()
              spin 1100ms → revealResult()
              buildMatrix → buildVisualOrder → stopAtPosition(1)
              showResultUi() → paylineView.showWinLines()
```

## 12. Что должен реализовать сервер

1. Хранить идентичный `BASE[84]` стрип.
2. Применять те же сдвиги: CENTER = rotate(BASE, 28), RIGHT = rotate(BASE, 56).
3. При получении ставки бросить `random(0, 83)` для каждого барабана.
4. Построить матрицу `buildMatrix(p0, p1, p2)`.
5. Проверить все 5 paylines по `resolveWin`.
6. Вернуть в Memo: `SLOT:{p0},{p1},{p2}:{WIN|LOSE}`.
